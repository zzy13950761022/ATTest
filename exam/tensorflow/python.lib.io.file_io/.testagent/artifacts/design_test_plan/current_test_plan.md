# tensorflow.python.lib.io.file_io 测试计划

## 1. 测试策略
- 单元测试框架：pytest
- 隔离策略：使用mock隔离底层C++ _pywrap_file_io模块和os.path函数，避免真实文件系统依赖
- 随机性处理：无随机性要求，使用固定测试数据

## 2. 生成规格摘要（来自 test_plan.json）
- SMOKE_SET: CASE_01, CASE_02, CASE_03, CASE_04, CASE_05, CASE_06
- DEFERRED_SET: CASE_07, CASE_08, CASE_09, CASE_10, CASE_11, CASE_12
- group列表与active_group_order: G1(文件存在性与状态查询), G2(文件读写操作), G3(目录与文件操作)
- 断言分级策略：首轮使用weak断言（类型检查、基础功能），最终轮启用strong断言（精确匹配、性能边界）
- 预算策略：S级用例60-70行，M级用例80-90行，参数不超过6个

## 3. 数据与边界
- 正常数据集：标准文件路径、文本内容、二进制数据
- 随机生成策略：不适用，使用固定测试用例
- 边界值：空字符串路径、空文件内容、空目录、超长路径（模拟）
- 负例与异常场景：
  - 不存在的文件/目录操作
  - 权限不足访问
  - 无效URI方案
  - 无效文件模式
  - 覆盖冲突
  - 递归删除验证

## 4. 覆盖映射
- TC-01 (CASE_01): 需求-必测路径1（文件存在性检查）
- TC-02 (CASE_02): 需求-必测路径5（文件统计信息）
- TC-03 (CASE_03): 需求-必测路径2（文件读写一致性）
- TC-04 (CASE_04): 需求-必测路径2（FileIO类读写）
- TC-05 (CASE_05): 需求-必测路径3（目录操作）
- TC-06 (CASE_06): 需求-必测路径4（文件复制）

- 尚未覆盖的风险点：
  - 云存储系统特定功能
  - 大文件分块读写性能
  - 并发文件访问竞争条件
  - 符号链接处理
  - 跨平台路径分隔符细节
